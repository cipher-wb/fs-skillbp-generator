# AI技能编辑器管线：需求梳理与共识确认书

这是一份用于对齐目标的复盘与需求确认文档，请仔细审阅。只有在我们对以下三点完全达成一致后，我才会进行下一步。

---

## 🎯 一、 我们的终极需求（愿景）

**核心痛点：**
策划配置一个技能往往需要在节点编辑器里拖拽大量节点、查阅表单录入魔法数字（特效ID、BuffID），步骤繁琐且容易出错。

**最终目标：**
实现**“自然语言到可用技能配置的自动化转换”**。
策划只需要输入类似于“*前摇10帧，在目标脚下放一个60帧的火圈，命中附带1001号点燃Buff*”的自然白话描述，AI 就能自动翻译为 `NodeEditor` 能够无损读取的节点系统，**在保持哈希值、节点 ID 以及 C# 关联不报错的前提下，可直接在游戏内测试运行。**

## 📁 二、 当前我所拥有的“知识”与文件

目前我们的工作空间内有以下核心资源，这是 AI 唯一能感知项目运行规律的地方：

1. **`TableAnnotation.json`（罗塞塔石碑）**
   - **它的作用：** 将难以理解的魔法数字（比如：`EnumValue:11`，`EnumValue:4`）翻译为了能看懂的各种指令（比如：“播放特效”、“添加buff”），并且规定了每个指令下的 `Params` 数组的格式（如 `Params[0]` 是特效ID，`Params[3]` 是持续时长）。
2. **`SkillGraph_302901...json` 等《宗门技能》范例库**
   - **它的作用：** 这些是人工配置的、且被引擎承认的复杂成功样例。能展示真实的连招设计、以及数学计算（如计算目标 XY 坐标再偏移的沙盒测试）。
3. **`SkillGraph_320004_躯壳.json`（容器底座）**
   - **它的作用：** 这是一个在编辑器里面 `Ctrl+S` 生成的空白底座。它自带了 `TableTash`（关键的序列化防腐校验码）和唯一的 `Config2ID`。它是 AI 注入节点的终极容器，用来欺骗和绕过 Unity 极度严格的文件校验拦截机制。

## 🛠 三、 AI 目前采用且失败的处理方案（复盘）

**我刚才干了什么（导致了失败）：**
1. 我读取了你的“造火圈、挂Buff”需求。
2. 我去翻了 `TableAnnotation.json`，找到了 `延迟执行(2)`、`播放特效(11)` 和 `添加buff(4)` 这三个节点的枚举。
3. 我用这三个节点组建了 JSON 数据块（分别命名为 ID 32000401, 32000402 等）。
4. 我通过 `edges` 数组，将它们按 `入口节点 -> 延迟 -> 特效 -> Buff` 的顺序连接起来。
5. 我将这些强行塞回了 `躯壳.json` 文件中。

**为什么这个“表面对的方案”会是“完全不可用的逻辑”：**
因为技能系统的 **真实触发逻辑并非单纯的线性连接。** 技能编辑器可能存在其独特的框架规则，例如：
- 特效和 Buff 是不能直接“串联”的（比如它们可能都需要连接到核心的释放判定节点，或者由父级节点派发）。
- `PackedParamsOutput` 的连线端口（`inputPortIdentifier`, `outputPortIdentifier`）可能需要非常严密的对应，稍微填错一点，参数就不互通。
- 特效如果是在目标脚下生成，不是单单填一个时间参数就能搞定的，可能必须带有“目标获取”或“位置解算”的上游控制节点才能生效。

## 📝 四、 请你定夺：下一步我们要确认什么？

在我们达成一致之前，我绝非为了写代码而写代码。为了保证生成的逻辑“懂你们的框架”，我需要你给我以下指导：

**问题 1：关于“正确逻辑”的定义**
既然我刚刚直接把（延迟->特效->Buff）串联起来是**无效**的，那么：在你们的编辑器里，一个标准包含【前摇+放特效的圈+给圈内人加Buff】的拓扑，**最少应该是什么样的节点结构？** 你是否能用文字简述一下标准规范？（比如：它需要一个特定的 `SkillExecuteNode` 统筹？还是走 `顺序执行` 节点分支？）

**问题 2：是否放弃原子节点瞎拼，改走“大模板复用”路线？**
就像我在上一份工业报告里提到的，**让 AI 直接从微小节点连线（甚至算坐标的连线），出 Bug 的概率极大**。
我们是否要先在项目内确定一个人工搭建好、可以工作的包含以上逻辑的“复合技能骨架/标准库的 Json”，发给 AI，**只让 AI 聪明地替换骨架里的 ID 和 帧数 参数**？

我对底层架构的规则理解还有偏差。如果你认同我们目前对需求和资源的梳理，请告诉我**“正确的火圈技能在编辑器里面至少连成什么样的拓扑图才是对的”**，或者给我**发一个已经配过正确火圈连线的参考 JSON 文件。**我们重新对焦后再继续！
