## 🔍 二、 历史问题复盘：为什么以前别人的AI总会出逻辑问题？

在没有 `TableAnnotation` 约束和没有理解底层运行态时，以往 AI 直接生成 JSON 会栽在以下“三座大山”：

1. **“黑盒枚举”与“魔法数字”导致的瞎编**
   以前的 AI 知道需要生成一个“添加 Buff”的动作，但它并不知道该动作对应的内部指令码是 `4 (TSET_ADD_BUFF)`，更不知道 `Params[2]` 代表 BuffID，`Params[3]` 代表持续帧数。因此 AI 要么强行伪造英文键值对（如 `"duration": 60`），要么填错位置导致引擎读取报错。
2. **强哈希隔离（TableTash 墙）与 GUID 冲突**
   Unity 编辑器底层具有严格的数据防腐机制，一旦检查到由 AI 无中生有生成的野鸡节点没有正确的 `TableTash`（哈希版本号）或跟现有库产生 `Config2ID` 重叠，直接引发反序列化异常并把数据丢弃。
3. **坐标散乱与拓扑解耦**
   真实的 JSON 中连线结构（`edges`）与数据结构往往是分离的。直接要求 AI 管理庞杂的多级 GUID 连线树，极易导致“断链”或死循环。

## ⚙️ 三、 工业化落地与高质量生成策略

要实现稳定、不出错的工业化自动生成，我们需要抛弃以往的暴利拼接，转入**“降维植入（注入）”**流程。

### 核心解法：白盒躯壳注入法 (Shell Injection)
这是目前最推崇且无痛的零成本代码运作模式，**步骤如下：**

1. **准备底座（去哈希痛点）**：策划在 Unity 技能编辑器里创建一个**全新的空白技能**（只需包含基础 SkillConfigNode 和几个空的占位 Node，什么都不连），只做一件事：**保存**。此时生成了一个拥有合法哈希和最新数据结构的`.json`“躯壳”文件。
2. **智能映射与语义装载**：AI 借助内置的 `TableAnnotation` 参数字典进行编译：
   - 策划输入：“发一枚1001子弹，带2005Buff”
   - AI 直接查表得知 -> 添加节点 `EnumValue:8 (创建子弹)`，将 `Params[X]` 填充为 1001；添加节点 `EnumValue:4 (添加Buff)`，挂载至 `Params[Y]`。
3. **精准融合**：AI 将计算正确的数据，精准替换躯壳文件中的 `ConfigJson` 与 `edges`。保持哈希值不动。
4. **引擎无缝读取**：策划切回引擎，实现所见即所得。

通过这种“你造容器我装酒”的模式，可以 100% 避免逻辑崩坏、参数乱套。

## ♻️ 四、 全策划团队复用的方法论论证

如果你想将这套系统封装为工具链普及给所有不熟悉后端的执行策划，不能靠教他们写 Prompt，而是要构建一套工业化的“提示词工作台”：

### 1. 提供专属的 GPTs / Agent 挂载点
不要让其他策划发纯文本。可以为团队统一部署一个技能定制版的 AI Agent。在 Agent 的 `System Instructions` 中预置微缩版的 `TableAnnotation.json`（去除无用信息，抽离出如 `Param: 1=顺序执行, 4=加buff` 等简明字典）。
其他自然语言交互只需要一键触发：“帮我把这段文本转成符合 C# 节点描述的结构”。

### 2. 通过“Skills（技能模板树）”而非零散节点来进行复用
这是一个极其核心的问题。**是否以 Skills 结构进行？答案是肯定的，且需要拔高到“复合子图模板（Templates/Prefabs）”的高度。**

从 `宗门技能流心法.json` 以及 `测试蓝图` 可以看出，一个复杂的技能（如附带随机弹道偏移的技能）包含了庞大的连线逻辑、各种数学节点的算式（正余弦、距离）。
如果期待每次 AI 都从最底层的（随机数+属性读取+数学节点）从零推导这些逻辑，这绝对会导致生成质量不可控！

**正确（工业化）的复用做法：**
1. **沉淀标准库**：你们已经在 `SkillEditor\Saves\Jsons\宗门技能` 中配好了极高水平的核心技能。从中提炼出标准的主干（如“标准突进斩击版”、“召唤物自爆阵法版”）。
2. **AI 从 "微观积木拼装" 升级为 "宏观模块填参"**：
   未来，别的策划说“我要一个剑客冲刺放毒气”，AI 不是去傻傻拼凑底层 XY 坐标位移计算器。而是直接锁定：调用“宗门套件的冲刺外壳（ID: XXXX）”，然后让 AI 的侧重点只放在将“毒气”的参数、Buff 的时长精确查字典配置进去。
3. **建立 Skills 语言规范 (DSL)**：统一大家描述技能的切入点（例如强制要求大家必须说清楚 前摇、判定方式、特效挂载点、执行效果、受击状态），规范自然语言输入，能减少 AI 70% 的推理错误。
